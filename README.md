# 🎓 Student Record Management System
**A robust database application built in C for efficient data handling and persistence.**

---

### 📖 Project Overview
This project is a console-based management system designed to handle academic records with high reliability. It focuses on core software engineering principles: **CRUD operations** (Create, Read, Update, Delete) and **Data Persistence** through file handling.

### 🛠️ Tech Stack & Skills
* **Language:** C
* **Logic:** Struct-based data modeling and Modular Programming
* **Storage:** File I/O for persistent data management
* **Deployment:** Interactive documentation and run guide hosted on **Netlify**

### 🚀 Key Technical Features
* **Data Modeling:** Uses C `structs` to define complex student attributes such as Name, Roll Number, and Academic Marks.
* **File Persistence:** Implements automated file handling to ensure records are saved permanently to the disk, preventing data loss between sessions.
* **Input Validation:** Built-in logic to handle incorrect data types and prevent system crashes during user interaction.
* **Modular Architecture:** The codebase is organized into clean, functional blocks for improved maintainability and readability.
